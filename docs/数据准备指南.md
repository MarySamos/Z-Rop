# 📊 数据准备指南

> 本文档说明如何为 BankAgent-Pro 项目准备数据

## 🎯 为什么需要导入数据到数据库？

你可能会问：**"我直接把 CSV 文件放到项目里不就行了吗？"**

简短回答：**不行**，因为：

```
┌─────────────────────────────────────────────────────────────────┐
│  用户问: "30岁以下客户有多少人？"                                  │
│                         ↓                                       │
│  LLM 生成 SQL: SELECT COUNT(*) FROM marketing_data WHERE age<30 │
│                         ↓                                       │
│  执行 SQL → 需要数据在 PostgreSQL 数据库中！                      │
│                         ↓                                       │
│  返回结果: "共有 8,234 人"                                       │
└─────────────────────────────────────────────────────────────────┘
```

**核心原因**：Text-to-SQL 功能需要将自然语言转换为 SQL 查询，而 SQL 只能在数据库中执行。

## 📥 数据准备步骤

### 步骤 1: 下载数据集

从 Kaggle 下载银行营销数据集：

🔗 **下载链接**: [Bank Marketing Dataset](https://www.kaggle.com/datasets/henriqueyamahata/bank-marketing)

下载后你会得到这些文件：
- `bank-additional-full.csv` ⭐ **推荐使用** (41,188 条完整数据)
- `bank-additional.csv` (10% 样本，4,119 条)
- `bank-full.csv` (旧版本)
- `bank.csv` (旧版本样本)

### 步骤 2: 放置数据文件

将下载的 CSV 文件放入项目目录：

```
BankAgent-Pro/
└── data/
    └── raw/
        └── bank-additional-full.csv   ← 放在这里
```

### 步骤 3: 确保数据库已就绪

1. 确保 PostgreSQL 服务正在运行
2. 确保已创建 `bank_agent` 数据库
3. 确保 `.env` 文件配置正确：

```env
DATABASE_URL=postgresql://postgres:你的密码@localhost:5432/bank_agent
```

### 步骤 4: 运行数据导入

**方式一：一键脚本（推荐）**

```powershell
cd backend
python scripts/setup_data.py
```

这个脚本会引导你完成：
1. ✅ 检查数据库连接
2. ✅ 初始化表结构
3. ✅ 导入 CSV 数据
4. ⭐ (可选) 导入 PDF 知识文档
5. ⭐ (可选) 训练 ML 模型

**方式二：分步执行**

```powershell
cd backend

# 1. 初始化数据库表
python scripts/init_db.py

# 2. 导入 CSV 数据
python scripts/import_csv.py

# 3. (可选) 导入 PDF 文档
python scripts/ingest_pdf.py docs/

# 4. (可选) 训练 ML 模型
python scripts/train_model.py
```

## 📊 数据集说明

### 字段说明（银行营销数据集）

| 字段 | 类型 | 说明 |
|------|------|------|
| `age` | 整数 | 客户年龄 |
| `job` | 字符串 | 职业类型 (admin, technician, ...) |
| `marital` | 字符串 | 婚姻状况 (married, single, divorced) |
| `education` | 字符串 | 教育程度 |
| `default` | 字符串 | 是否有违约记录 (yes/no) |
| `housing` | 字符串 | 是否有住房贷款 |
| `loan` | 字符串 | 是否有个人贷款 |
| `contact` | 字符串 | 联系方式 (cellular, telephone) |
| `month` | 字符串 | 最后联系月份 |
| `day_of_week` | 字符串 | 最后联系星期 |
| `duration` | 整数 | 通话时长（秒） |
| `campaign` | 整数 | 本次活动联系次数 |
| `pdays` | 整数 | 距离上次联系天数 |
| `previous` | 整数 | 之前活动联系次数 |
| `poutcome` | 字符串 | 之前营销结果 |
| `emp.var.rate` | 浮点数 | 就业变动率 |
| `cons.price.idx` | 浮点数 | 消费者物价指数 |
| `cons.conf.idx` | 浮点数 | 消费者信心指数 |
| `euribor3m` | 浮点数 | 3个月期欧元银行间利率 |
| `nr.employed` | 浮点数 | 就业人数 |
| `y` | 字符串 | **目标变量**：是否订阅定期存款 (yes/no) |

### 示例查询

导入数据后，你可以用以下 SQL 测试：

```sql
-- 查看数据总量
SELECT COUNT(*) FROM marketing_data;

-- 按职业统计转化率
SELECT job, 
       COUNT(*) as total,
       SUM(CASE WHEN y='yes' THEN 1 ELSE 0 END) as converted,
       ROUND(100.0 * SUM(CASE WHEN y='yes' THEN 1 ELSE 0 END) / COUNT(*), 2) as rate
FROM marketing_data
GROUP BY job
ORDER BY rate DESC;

-- 年龄分布
SELECT 
    CASE 
        WHEN age < 30 THEN '30岁以下'
        WHEN age < 40 THEN '30-40岁'
        WHEN age < 50 THEN '40-50岁'
        ELSE '50岁以上'
    END as age_group,
    COUNT(*) as count
FROM marketing_data
GROUP BY age_group;
```

## 🔧 常见问题

### Q1: 导入时报错 "无法连接数据库"

**解决方案**：
1. 检查 PostgreSQL 服务是否启动
2. 检查 `.env` 中的 `DATABASE_URL` 是否正确
3. 确保数据库 `bank_agent` 已创建

### Q2: CSV 文件格式不对

**解决方案**：
- 脚本会自动检测分号 `;` 或逗号 `,` 分隔符
- 如果仍有问题，请确保文件是 UTF-8 编码

### Q3: 数据量太大导入很慢

**解决方案**：
- 可以先用 `bank-additional.csv`（10% 样本）测试
- 完整版 41,188 条数据导入通常在 5-10 秒内完成
