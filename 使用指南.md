## 项目概述

BankAgent-Pro 是一个智能银行数据分析平台，结合了大语言模型、数据科学和机器学习技术，提供以下核心功能：

### 🎯 核心特色

1. **用户认证系统** - JWT + RBAC 权限管理
2. **数据工程** - CSV/Excel 数据导入、PDF 文档向量化
3. **智能分析** - LangGraph 智能体（待实现）
4. **数据可视化** - Pyecharts + ECharts（待实现）
5. **深度数据分析** - 统计分析、聚类、关联挖掘（待实现）

### 🛠 技术栈

**后端**
- FastAPI + Uvicorn（Web 框架）
- PostgreSQL + pgvector（数据库）
- SQLAlchemy（ORM）
- LangChain + LangGraph（LLM 智能体）
- Pandas + NumPy + Scikit-learn（数据处理）

**前端**
- Vue 3 + Vite
- Element Plus（UI 组件）
- ECharts（图表库）

## 快速开始

### 1️⃣ 安装依赖

```bash
# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
# Windows:
.venv\Scripts\activate
# Linux/Mac:
source .venv/bin/activate

# 安装依赖包
pip install -r requirements.txt
```

### 2️⃣ 配置环境变量

创建 `.env` 文件（项目根目录）：

```env

APP_VERSION=1.0.0
DEBUG=True

# 数据库配置
DATABASE_URL=postgresql://postgres:your_password@localhost:5432/bank_agent

# JWT 配置
SECRET_KEY=your-secret-key-change-this-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30

# OpenAI API 配置（使用阿里云 DashScope）
OPENAI_API_KEY=your-openai-api-key
OPENAI_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1

# 智谱 API（可选）
ZHIPU_API_KEY=
ZHIPU_BASE_URL=

# LangSmith（可选）
LANGSMITH_PROJECT=
LANGSMITH_API_KEY=
```

**⚠️ 重要提示**：
- 请修改 `SECRET_KEY` 为随机字符串（生产环境）
- `OPENAI_API_BASE` 可根据使用的 LLM 服务商调整

### 3️⃣ 数据库设置

#### 安装 PostgreSQL

**Windows**:
1. 下载安装包：https://www.postgresql.org/download/windows/
2. 安装时记住设置的密码
3. 确保服务已启动

```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
```

```bash
CREATE EXTENSION IF NOT EXISTS vector;  -- 可选，用于向量相似度搜索
```

## 数据库设置

### 数据表结构

#### 1. users（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| employee_id | String(50) | 工号（唯一） |
| name | String(100) | 姓名 |
| department | String(100) | 部门 |
| hashed_password | String(255) | 密码哈希 |
| role | String(50) | 角色（user/admin/analyst） |
| is_active | Boolean | 是否激活 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

#### 2. marketing_data（银行营销数据表）

存储银行营销活动数据，包含：
- 客户基本信息（年龄、职业、婚姻状况、教育程度）
- 联系方式（联系方式、最后联系时间）
- 营销活动信息（活动次数、通话时长）
- 经济指标（就业变动率、消费者物价指数等）
- 目标变量（是否订阅定期存款）

#### 3. knowledge_docs（知识文档表）

存储 RAG 检索用的知识文档，包含：
- 文档标题、内容
- 文件路径和类型
- 向量嵌入（JSON 格式）
- 元数据

#### 4. data_tables（数据表元数据管理）

管理系统中数据表的信息

#### 5. operation_logs（操作日志审计表）

记录用户的所有操作行为

### 开发流程

#### 1. 添加新的 API 接口

**步骤**：

1. 在 `app/schemas/` 定义请求/响应模型
2. 在 `app/services/` 编写业务逻辑
3. 在 `app/api/endpoints/` 创建路由
4. 在 `app/api/api_v1.py` 注册路由

**示例**：

```python
# 1. schemas/task.py
from pydantic import BaseModel

class TaskCreate(BaseModel):
    title: str
    description: str

class TaskResponse(BaseModel):
    id: int
    title: str
    description: str

# 2. services/task_service.py
def create_task(db: Session, title: str, description: str):
    task = Task(title=title, description=description)
    db.add(task)
    db.commit()
    db.refresh(task)
    return task

# 3. endpoints/task.py
from fastapi import APIRouter, Depends
from sqlalchemy.orm import Session
from app.core.database import get_db

router = APIRouter(prefix="/tasks", tags=["任务管理"])

@router.post("/", response_model=TaskResponse)
def create_task(task: TaskCreate, db: Session = Depends(get_db)):
    return create_task(db, task.title, task.description)

# 4. api_v1.py
from app.api.endpoints import task
api_router.include_router(task.router)
```

#### 2. 使用权限控制

```python
from app.api.dependencies import get_current_user, require_admin

# 需要登录
@router.get("/profile")
def get_profile(current_user: User = Depends(get_current_user)):
    return current_user

# 需要管理员权限
@router.delete("/users/{user_id}")
def delete_user(user_id: int, admin: User = Depends(require_admin)):
    # 只有 admin 角色可以访问
    pass

# 需要分析师权限（admin 或 analyst）
@router.post("/analysis")
def analyze_data(analyst: User = Depends(require_analyst)):
    # admin 或 analyst 角色可以访问
    pass
```

#### 3. 数据库操作

```python
from sqlalchemy.orm import Session
from app.core.database import SessionLocal
from app.db.models import User

# 使用依赖注入（推荐）
```

## 常见问题

### Q1: 数据库连接失败

**错误信息**：
```
sqlalchemy.exc.OperationalError: (psycopg2.OperationalError) could not connect to server
```

**解决方案**：

1. 检查 PostgreSQL 服务是否启动
2. 检查 `.env` 中的 `DATABASE_URL` 是否正确
3. 确认数据库已创建：`psql -U postgres -c "CREATE DATABASE bank_agent;"`

### Q2: JWT Token 验证失败

**错误信息**：
```
HTTPException: 401 Unauthorized - 无法验证凭据
```

**解决方案**：

1. 检查 Token 是否过期（默认 30 分钟）
2. 确认请求头格式：`Authorization: Bearer <token>`
3. 检查 `.env` 中的 `SECRET_KEY` 是否一致

### Q3: PDF 导入失败

**错误信息**：
```
❌ 错误: PDF 读取失败
```

**解决方案**：

1. 确认 PDF 文件未加密
2. 检查 PDF 是否包含文本（扫描版需要 OCR）
3. 尝试使用其他 PDF 重新生成

### Q4: 依赖安装失败

**错误信息**：
```
error: Microsoft Visual C++ 14.0 is required
```

**解决方案**：

**Windows**：
1. 安装 Microsoft C++ Build Tools
2. 或使用预编译的 wheel 文件

**Linux**：
```bash
sudo apt install python3-dev build-essential
```

### Q5: pgvector 扩展安装失败

**警告信息**：
```
⚠️ pgvector 扩展未安装
```

**说明**：这不影响项目运行，向量数据将存储为 Text 格式

**可选安装**：

```bash
# Ubuntu/Debian
git clone --branch v0.5.0 https://github.com/pgvector/pgvector.git
cd pgvector
make
sudo make install
# 安装后重启 PostgreSQL
sudo systemctl restart postgresql
```

## 生产部署

### 1. 环境变量检查清单

```env
DEBUG=False  # 必须设为 False
SECRET_KEY=<使用强随机字符串>
DATABASE_URL=<生产数据库连接>
OPENAI_API_KEY=<生产 API Key>
```

### 2. 依赖安装

```bash
pip install -r requirements.txt
```

### 3. 启动服务

**使用 Gunicorn（推荐）**：

```bash
pip install gunicorn
gunicorn backend.main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
```

**使用 Uvicorn**：

```bash
uvicorn backend.main:app --host 0.0.0.0 --port 8000 --workers 4
```

### 4. 使用 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```
